# AI Footprint — VS Code Extension

**In-editor attribution overlays for AI-generated code.**

Detect, attribute, and visualise AI-generated code directly inside VS Code. Works with your local [AI Footprint](https://github.com/ai-footprint/ai-footprint) snippet registry — no cloud, no telemetry.

## Features

| Feature | Description |
|---------|-------------|
| **Inline overlays** | Colour-coded attribution badges on AI-generated lines showing model, confidence, and similarity |
| **Diagnostics** | AI detections appear in the Problems panel with configurable severity |
| **CodeLens** | Clickable annotations above matched lines |
| **Status bar** | Live match count in the status bar |
| **Report panel** | Full HTML report with stats, match table, and confidence breakdown |
| **Auto-scan on save** | Files are scanned automatically when saved (configurable) |

## Detection Engine

The extension uses the same 4-tier detection pipeline as the CLI:

1. **Exact hash matching** — SHA-256 comparison against registered snippets
2. **Fuzzy matching** — N-gram Jaccard similarity (token + structural, 0.4/0.6 weighted)
3. **AST matching** — Structural shape comparison with LCS sequence similarity
4. **Pattern heuristics** — Regex detection of AI markers (`// generated by`, `@ai-generated`, `copilot`, etc.)

Supported languages: TypeScript, JavaScript, Python, Go, Rust, Java, Kotlin, C/C++, C#, Swift, Vue, Svelte, Astro, Shell scripts.

## Commands

Open the Command Palette (`Cmd+Shift+P` / `Ctrl+Shift+P`) and type "AI Footprint":

| Command | Description |
|---------|-------------|
| `AI Footprint: Scan Workspace` | Scan all code files in the workspace |
| `AI Footprint: Scan Current File` | Scan the active editor |
| `AI Footprint: Toggle Attribution Overlay` | Show/hide inline attribution badges |
| `AI Footprint: Show Report` | Open the HTML report panel |

## Settings

| Setting | Default | Description |
|---------|---------|-------------|
| `aiFootprint.enableOnSave` | `true` | Automatically scan files when saved |
| `aiFootprint.fuzzyThreshold` | `0.6` | Minimum similarity score for fuzzy matching (0.0–1.0) |
| `aiFootprint.overlayEnabled` | `true` | Show inline attribution overlays |
| `aiFootprint.diagnosticSeverity` | `Information` | Severity level: `Error`, `Warning`, `Information`, or `Hint` |

## Installation

### From VSIX (local)

1. Build the extension:
   ```bash
   cd extension
   npm install
   npm run build
   npx @vscode/vsce package --allow-missing-repository
   ```

2. Install in VS Code:
   - Open **Extensions** sidebar (`Cmd+Shift+X`)
   - Click **⋯** → **Install from VSIX…**
   - Select `ai-footprint-vscode-0.1.0.vsix`

### Development (F5)

1. Open the `extension/` folder in VS Code
2. Run `npm install` in the terminal
3. Press **F5** to launch the Extension Development Host
4. Test commands via the Command Palette

## Prerequisites

The extension reads your snippet registry from `~/.ai-footprint/snippets.json`. Set it up with the CLI:

```bash
npx ai-footprint init
npx ai-footprint add-snippet --file mycode.ts --source "copilot" --model "gpt-4.1"
```

Or register an entire directory:

```bash
npx ai-footprint track ./src --source "claude" --model "claude-opus-4.6"
```

```
│  VS Code Editor                                 │
│  ┌───────────────────────────────────────────┐  │

│  │  ├─ triggers scan on save / on demand     │  │

│  ├───────────────────────────────────────────┤  │
│  │  scanner.ts                               │  │
│  │  ├─ loads ~/.ai-footprint/snippets.json   │  │
│  │  ├─ exact hash matching (SHA-256)         │  │
│  │  ├─ fuzzy matching (n-gram Jaccard)       │  │
│  │  ├─ AST matching (structural shapes)      │  │

│  ├───────────────────────────────────────────┤  │
│  │  diagnostics.ts → Problems panel          │  │
│  │  overlay.ts     → inline decorations      │  │
│  │  statusBar.ts   → match count badge       │  │
│  │  reportPanel.ts → HTML webview report     │  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
```

## Security

- **No network calls** — all detection runs locally
- **No telemetry** — zero data leaves your machine
- **Symlink protection** — symlinks are skipped during workspace scans
- **File size limits** — files over 100 MB are skipped to prevent OOM
